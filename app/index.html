Realtime Teacher Dashboard

Worked for

Junyo • Menlo Park, California • June 2014 - August 2014

The product

The realtime teacher dashboard is part of the Junyo Streams product. The dashboard provides teachers with a realtime view into what students are doing in their class. Teachers are notified when a student is behind or having trouble, and they can see the overall progress of all their classes down to individual students.

What I worked on

I worked on a team with 2 other developers to build the dashboard and accompanying backend components. We built an event store and data processing backend in Python using HBase for the fast layer and PostgreSQL for the data processing layer. The secure PII (Personally Identifiable Information) server was writting in Python using the Django framework. The dashboard itself (which is what I primarily worked on) was written in Javascript using React for the dynamic UI and the FLUX architecture for simplified data flow, modular code, and ease of testing.

Other technologies used

Jest • unit testing
Less • css styles
Gulp • automation, asset revving, code minification, preprocessing, and style sheet compilation
Browserify • modularize Javascript code
Git + Github • version control and collaboration

Chemical Safety App

Interned for

DHI • Hørsholm, Denmark • June 2013 - July 2013

The problem

DHI's chemical engineers found it cumbersome to carry around hefty, oversized Material Safety Data Sheets (MSDS) and wanted to be able to browse through the company's collection of chemical safety sheets quickly and easily on their smart phones.

The solution

I wrote a mobile app prototype in JavaScript using the Appcelerator Titanium SDK and Alloy MVC framework . This combination of frameworks allowed me to get up and running quickly on Android, iOS, and Windows Phone with a single code base. I also wrote a django + PostgreSQL database backend to store, access, and modify the data sheets through a REST API and user-friendly admin interface.

Sleep Survey Web App

Interned for

41Winks • Menlo Park, California • July 2013 - August 2013

The problem

Nearly 70 million Americans suffer from chronic sleep disorders like sleep apnea, insomnia, restless leg syndrome, and narcolepsy. [1]

The solution

I wrote a survey web app for patients and doctors to diagnose potential sleep disorders. A number of Stanford researchers use the app to gather data for further study. The frontent was developed in HTML/CSS/JS with AngularJS as the MV* framework and Topcoat for basic mobile styles. The backend was developed in Python using the Flask web framework.
